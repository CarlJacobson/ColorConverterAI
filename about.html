FAQ

What value should I use for the number of colors to extract?

Generally, higher values of colors to extract are good to use when working with realistic photos or paintings with a high total amount of colors in the image. Lower number of colors to extract are good for low dynamic range images such as cartoons, digitally generated images, etc. There is no hard rule for the number of colors you should extract from an image. You should play around with the slider to get an idea of what looks good for your use case.

Why do my output images look bad?

Output images can look bad for a variety of reasons. First thing to check would be the number of colors to extract.

Are my images uploaded anywhere?

No. All processing happens locally in your web browser using JavaScript. Your images are never sent to a server.

Definitions

Target Image

The Target Image is the image that you want to change the color palette on. It can be any jpg or png file. Generally works best with cartoon/animated images with limited color ranges

Palette Image

The Image you want to extract your color palette from. It can be any jpg or png file. Generally works best with photographs or paintings with a large range of colors. The palette image can be used with either the Palette Extractor, or Palette Transfer function. The Palette Extractor tool takes in the palette image, and outputs a downloadable png file with the color palette of the image with the associated hex codes. This could be useful for graphic design or general reference. If you decide you want to apply the extracted palette to another image, you can use the "Palette Transfer" link in the top nav. The image and number of colors to extract should be saved across page changes.

How does it work?

ColorConverter.AI converts images using a Kmeans unsupervised machine learning algorithm. An image is comprised of its individual pixels. Each individual pixel has a Red, Green, and Blue (RGB) value ranging from 0 to 255. These RGB values can be combined in any proportion to make virtually any color. The RGB values can be thought of as coordinates in a 3d “color space”, with the RGB values on X, Y, and Z axes respectively. The k means algorithm takes both a list of the palette image RGB values and “number of colors to extract” (k) as its input values. It then iteratively calculates the “k” points in the 3d color space that best summarize the image's colors. The final list of k number of coordinates (“centroids”) is then stored as the color palette of that image

To apply that color palette to another image, each pixel in the target image is iterated over. The distance between that point, and each centroid is then calculated. The pixel is assigned to whichever centroid it is closest to.
